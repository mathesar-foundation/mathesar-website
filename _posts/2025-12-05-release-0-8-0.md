Mathesar 0.8.0 is here! This release makes it easier to work with connected data by introducing **related columns**. You can now extend a table to show columns from related tables directly in the table page. This release also adds **nested filtering for more complex queries** and support for **working with database views**.

Additional improvements include configurable SSL for database connections and an improved experience on phones and tablets.

### ğŸ”— Extend tables with Related columns

![Extending the Movies table with genres from a related table](/images/blog/0.8.0/extend-movies-with-genres.png)

Use the "Extend" button to display columns from related tables into your current table view. After displaying the columns, you can edit them by adding or removing values. You'll see a dropdown of available choices with checkboxes to add or remove them from the record.

This feature currently only supports "many to many" relationships, like `genres` related to `movies` via a `movies_genres` table, or `categories` applied to `support_tickets` via a `ticket_categories` table.

![Complex nested filters with AND/OR groups](/images/blog/0.8.0/complex-filters.png)

Filters now support nested groups with AND, OR, and NOT logic, allowing you to build more expressive and flexible queries. You can group related conditions together and then combine those groups to find records that match one set of criteria or another.

![Views appear on the schema page with a distinct icon and tag](/images/blog/0.8.0/views-on-schema-page.png)

![Viewing data from a database view in Mathesar](/images/blog/0.8.0/example-view.png)

Database views now appear in Mathesar alongside tables. Views are clearly labeled with an icon and blue color so you can distinguish them from editable tables, while still navigating and inspecting view data inside of Mathesar.

### ğŸ”’ Configurable SSL mode for database connections

![SSL mode configuration in database settings](/images/blog/0.8.0/db-ssl-settings.png)

You can now choose an SSL mode when connecting to external databases in the Mathesar UI. You can also set the SSL mode for internal database connections via environment variables. This improves compatibility with databases that require SSL and gives you more control over how connections are secured.

### ğŸ” Column-level UPDATE privilege support

Mathesar now respects PostgreSQL's column-level UPDATE privileges. If your database permissions allow updates to only certain columns, you can edit those columns in Mathesar even if you don't have table-level UPDATE privileges.
